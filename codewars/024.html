Hamming Numbers (4 kyu)

A Hamming number is a positive integer of the form 2i3j5k, for some non-negative integers i, j, and k.

Write a function that computes the nth smallest Hamming number.

Specifically:

  • The first smallest Hamming number is 1 = 203050
  • The second smallest Hamming number is 2 = 213050
  • The third smallest Hamming number is 3 = 203150
  • The fourth smallest Hamming number is 4 = 223050
  • The fifth smallest Hamming number is 5 = 203051

The 20 smallest Hamming numbers are given in example test fixture.

Your code should be able to compute all of the smallest 5,000 (Clojure: 2000, NASM: 13282) Hamming numbers without timing out.

Starting Code

def hamming(n):
  """Returns the nth hamming number"""

My Solution

import itertools


def hamming(n):
    # We will be determining and storing all of the first n Hamming numbers in order.
    # Pre-allocation seems reasonable.
    # (At any time, for each multiplier, we will only actually need a limited subsequence to which we can add at one end 
    # and from which we can draw at the other, which is why deques/queues are a sensible option too.)
    hamming_sequence = [None] * n
    
    # With zero-indexing throughout, the 0th Hamming number is 1.
    # new_element will always represent the most recently determined element.
    hamming_sequence[0] = new_element = 1
    
    # Each element in the sequence after the 0th is 2 times, 3 times, or 5 times some preceding element.
    
    # These iterators will allow us to advance along the sequence,
    # separately identifying the next element to double, the next element to triple, and the next element to quintuple.
    doubler_iterator, tripler_iterator, quintupler_iterator = itertools.tee(hamming_sequence, 3)
    
    # At the end of each iteration to be done, we determine the next element in the sequence as the smallest of 
    # the current doubled candidate, the current tripled candidate, and the current quintupled candidate.
    # On the next iteration, the winning candidates' iterators will be advanced and their values updated.
    
    # We are careful to not unnecessarily repeatedly access the same sequence element for multiplication by the same multiplier.
    # (Though note that different elements multiplied by different mutlipliers can certainly produce equal candidates.  
    # And when such tied candidates win an iteration, the winning value is added to the sequence just once, 
    # then the winning iterators advance identically and the corresponding candidates update differently.)    
    
    # We are also careful to compute only candidates that are immediately needed, 
    # rather than computing and accumulating a sequence of candidates that will never be used.
    
    # To get all three iterators started and the first true candidates generated in the first iteration, 
    # we initialize the values of all three candidates to 1, the first element in the sequence.
    doubled = tripled = quintupled = 1
    
    for new_element_index in range(1, n):
        if new_element == doubled:
            doubled = 2 * next(doubler_iterator)
        if new_element == tripled:
            tripled = 3 * next(tripler_iterator)
        if new_element == quintupled:
            quintupled = 5 * next(quintupler_iterator)

        # (While we are generally warned against touching an iterable once itertools.tee has been applied to it,
        # the ways in which we iterate over and modify hamming_sequence in this code should be just fine.)
        hamming_sequence[new_element_index] = new_element = min(doubled, tripled, quintupled)
    
    return new_element  # == hamming_sequence[-1] == hamming_sequence[n - 1]

My Notes

I figured that something very similar to the approaches I learned from solutions to the "Twice Linear" kata would work well here. I use a list rather than three deques/queues, and I make a few (I hope) minor improvements.
